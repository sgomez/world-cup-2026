# Public slug-based share route for community rankings

The platform is otherwise strictly members-only: non-members never see a Community's Bets, and the **Leaderboard** lives behind authentication. We now add the first **public, unauthenticated route** — `/{locale}/share/{slug}` — so a Community Member can post their ranking to x.com / Telegram. The route serves OpenGraph metadata and an autogenerated `1200×630` card (best 3 + worst 3 Bets by **label** and **Score**, titled with the Community name — never the Bet owner's name), then redirects any human visitor to the authenticated `/communities/{slug}` (logged-out users via `/login?callbackUrl=…`). It exposes **no** prediction content — only ranks, Bet labels, and points.

The link is keyed by the public **slug**, not a secret token. We accept that this makes the ranking *semi-public*: anyone who can guess or learns the slug sees the card. We judge this acceptable because the slug is derived from the Community name, and the Community name is itself only known to members and invitees — so reaching the share link is the same level of secret as knowing the Community exists. A separate share token was rejected as needless capability machinery for content the sharer is deliberately broadcasting anyway; reusing the **Invite Link** token was rejected because that grants *join* and would leak join rights to every x.com viewer.

The share card is excluded for **Imported Communities**: the button is hidden and the route returns 404. Imported Bet **labels** carry real participant names under **Label Obfuscation** (ADR 0023) and are owner-only — publishing them, even anonymized of owner, would defeat that rule.

The shared link carries a `?t={minute}` parameter: a minute-floored cache-buster set at share time. It is *not* a data filter — the card always renders the live Leaderboard at crawler-fetch time — but a new minute yields a new URL, prompting Telegram/x to re-fetch a fresh card while same-minute requests reuse the platform-cached image. The OG image is rendered with Next.js `ImageResponse` (Satori), in the Community's locale; **Provisional** status is not surfaced on the static card.

## Consequences

- The members-only-secrecy invariant now has one deliberate, documented hole. Any future change to Bet-label content or Leaderboard scope must reconsider this public surface.
- Slug-obscurity is the only access control on the route; there is no revocation. A Community whose name leaks has a permanently reachable ranking card until the feature or slug changes.
